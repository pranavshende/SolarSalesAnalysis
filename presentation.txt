SLIDE 1 / INTRODUCTION: The Problem & Our Stack
--------------------------------------------------
• The Problem: "Currently, data regarding India's solar capacity, state-wise installations, and revenue generation is highly fragmented. Government agencies publish static PDFs or raw Excel sheets, making it incredibly difficult for manufacturers, investors, and policymakers to track trends or forecast future demand."
• The Solution: "We built an integrated, AI-driven web platform that acts as a centralized 'Single Source of Truth'. It processes historical data, visualizes it interactively, and predicts future hardware requirements using Machine Learning."
• The Architecture: 
  - Frontend: React.js and Tailwind CSS for a responsive, modern UI.
  - Backend: Node.js and Express to handle RESTful APIs.
  - Database: PostgreSQL, allowing us to enforce relational integrity and aggregation queries.
  - AI/ML Layer: Python Fast-API microservice using Scikit-Learn for forecasting and computer vision.

--------------------------------------------------
SLIDE 2 / THE DATASET & DATA ENGINEERING
--------------------------------------------------
• Data Sources: "We compiled our raw data from government endpoints, resulting in two core files: 'Year & state Wise Data.xlsx' (13 KB) and 'city_wise_solar_2014_2024.csv' (2.6 KB)."
• Scale & Scope: "Combined, this yields over 500+ granular temporal capacity records tracking solar additions across 36 Indian States and Union Territories, plus hyper-local mega solar parks."
• Temporal Range: "The dataset spans exactly 10 years (2014 to 2024), which provides enough historical variance for our Machine Learning models to learn the long-term accelerating curvature of adoption."

SLIDE 6 / DEMONSTRATION: Deep Dive - AI Polynomial Forecasting Model
--------------------------------------------------
• Navigation: (Open ML Forecasting Engine)
• The Objective: "Historical data is useless without future context. We built a Python ML microservice to predict the exact capacity India will install from 2025 to 2030."
• The Algorithm - Polynomial Regression (Degree=2): 
  "Solar growth in India is not a straight line; it follows an exponential curve due to compounding investments and falling panel prices. A standard Linear Regression underfits this curve. Therefore, we engineered an AI Polynomial Regression Model (Degree 2) using Scikit-Learn's `PolynomialFeatures`. This allows the model to map the parabolic, accelerating trend of solar adoption perfectly."
• The Train/Test Split Methodology: 
  "Since we have a 10-year macro dataset (2014-2024), we utilize an 80/20 Train/Test Split logic to validate our model. We split the data temporally:
   1. Training Set (80%): The algorithm is trained on data from 2014 to 2021 to learn the curvature of the growth trend.
   2. Testing Set (20%): We test the model's predictions against the actual known data from 2022 to 2023. 
   3. Evaluation: We calculate the Root Mean Squared Error (RMSE) of the test set. If the Polynomial model outperforms the baseline Linear Regression, the system dynamically selects the Polynomial model."
• Generating the Forecast: 
  "Once validated via the train/test split, the model is retrained on the entire 10-year dataset and predicts the capacity targets up to 2030, generating dynamic Confidence Intervals (the shaded blue region on the graph)."
